## Push vs Pull Performance Analysis

### When pull (gather) wins

Pull assigns one thread per *destination* node: each thread reads from all incoming neighbors. Access to the destination's new rank is thread-local, so no atomic scatter is needed. This works well on uniform or road-network-like graphs such as roadNet-CA, where threads do relatively similar work and avoid atomic write contention.

### When push (scatter) loses

Push assigns one thread per *source* node: each thread writes to multiple destination ranks via `atomic.add`. On some power-law graphs, hub nodes cause load imbalance and popular destinations cause atomic contention, so push can lose badly, as on wiki-talk.

### When push wins

Push can win when the outgoing traversal has better locality and the atomic contention is lower than the cost of pull-side incoming traversal. In this run, push wins on `com-youtube`, `amazon0601`, and `soc-livejournal`.
